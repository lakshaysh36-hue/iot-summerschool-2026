# Q19. Difference between `analogWrite()` and `analogRead()`, and PWM

### `analogRead()`

`analogRead()` is used to read analog voltage from sensors connected to the Arduino's analog input pins (A0–A5). It converts the input voltage into a digital value between **0 and 1023**, where 0 represents 0 V and 1023 represents 5 V (on a typical Arduino Uno).

**Practical IoT Example:**
A smart irrigation system uses `analogRead()` to read the value from a soil moisture sensor. Based on the sensor reading, the system decides whether to turn the water pump on or off.

### `analogWrite()`

`analogWrite()` is used to generate a **PWM (Pulse Width Modulation)** signal on PWM-enabled pins (3, 5, 6, 9, 10, and 11). It accepts values from **0 to 255** to control the duty cycle, allowing devices such as LEDs and motors to operate at different intensity or speed levels.

**Practical IoT Example:**
A smart street lighting system uses `analogWrite()` to adjust the brightness of LEDs automatically depending on the surrounding light conditions.

### What is PWM?

**PWM (Pulse Width Modulation)** is a technique in which a digital output rapidly switches between HIGH and LOW. By changing the amount of time the signal stays HIGH (called the **duty cycle**), Arduino can simulate an analog output even though it only produces digital signals.

PWM is mainly used to control the amount of power delivered to devices such as LEDs, DC motors, and buzzers. It enables smooth control of brightness, motor speed, and other output levels without requiring true analog output hardware.

**Practical IoT Example:**
In a smart home lighting system, PWM is used to gradually increase or decrease the brightness of an LED based on ambient light sensor readings, helping to save energy while maintaining comfortable lighting.

# Q20. setup() and loop() Functions in Arduino

### setup()

The `setup()` function is executed **only once** when the Arduino is powered on or reset. It is used to initialize the program, such as configuring pin modes, starting serial communication, and setting up sensors or other hardware.

**Example:**


void setup() {
  pinMode(13, OUTPUT);
  Serial.begin(9600);
}


### `loop()`

The `loop()` function runs **continuously** after `setup()` finishes. It contains the main logic of the program, such as reading sensors, controlling LEDs or motors, and responding to user inputs.

**Example:**


void loop() {
  digitalWrite(13, HIGH);
  delay(1000);
  digitalWrite(13, LOW);
  delay(1000);
}


### Effect of Using a Long `delay()` in `loop()`

A long `delay()` pauses the entire program for the specified time. During this pause, the Arduino cannot read sensors, process button presses, or perform any other tasks. As a result, the system becomes less responsive and may miss important sensor readings or user inputs.

For example, if `delay(5000)` is used, the Arduino waits for 5 seconds before executing the next instruction, making it unable to respond immediately to changes.

### Non-Blocking Alternative

The recommended non-blocking alternative is to use the `millis()` function. Instead of stopping the program, `millis()` keeps track of elapsed time while allowing the Arduino to continue reading sensors and executing other tasks.

Using `millis()` improves responsiveness because multiple operations can run simultaneously without blocking the program, making it ideal for real-time IoT applications.
